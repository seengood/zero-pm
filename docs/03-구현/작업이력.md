- ✅ 기존 CPM 로직 회귀 테스트 통과

**알고리즘 상세**:

**Forward Pass (전진 계산)**:
```
For each task in topological order:
  ES = max(predecessor EF + lag based on dependency type)
  EF = ES + duration (in business days)
```

**Backward Pass (후진 계산)**:
```
For each task in reverse topological order:
  LF = min(successor LS - lag based on dependency type)
  LS = LF - duration (in business days)
  Total Float = LS - ES
  Critical = (Total Float == 0)
```

**Dependency 타입별 처리**:
- **FS (Finish-to-Start)**: Successor ES = Predecessor EF + lag
- **SS (Start-to-Start)**: Successor ES = Predecessor ES + lag
- **FF (Finish-to-Finish)**: Successor EF = Predecessor EF + lag
- **SF (Start-to-Finish)**: Successor EF = Predecessor ES + lag

### Event-Driven Architecture 구현

#### 1. Observer Pattern 기반 이벤트 시스템 구현
**완료 시간**: 2025-12-05 22:00

**작업 내용**:
- 중앙 이벤트 관리자 및 Observer 패턴 구현
- Task 업데이트 로직을 이벤트 기반으로 리팩토링
- 바 드래그 시 Duration 업데이트 문제 해결

**생성된 파일**:
1. `src/lib/taskEventEmitter.ts` - 중앙 이벤트 관리자
   - Publish/Subscribe 패턴 구현
   - 이벤트 타입: `task:created`, `task:updated`, `task:deleted`, `link:created`, `link:updated`, `link:deleted`
   
2. `src/lib/taskObservers.ts` - Observer 클래스들
   - `DBObserver`: 데이터베이스 저장 처리
   - `UIObserver`: UI 상태 업데이트 처리
   - `ScheduleObserver`: Successor 재계산 처리 (준비)

**수정된 파일**:
1. `src/components/GanttView.jsx`
   - Observer 초기화 및 이벤트 리스너 등록
   - `update-task` intercept 추가 (바 드래그 이벤트 처리)
   - `resize-task` intercept 추가 (바 크기 조정 이벤트 처리)
   - `diff` 값을 사용한 duration 계산 로직 구현
   - `handleTaskUpdate`를 이벤트 발생 방식으로 전환
   
2. `src/lib/dateUtils.ts`
   - UTC 날짜 파싱 개선 (`+00:00` 형식 지원)
   - Date 객체의 UTC 자정 감지 로직 추가
   - 타임존 변환 문제 해결

**구현된 기능**:
- ✅ 중앙 이벤트 관리 시스템 (TaskEventEmitter)
- ✅ Observer Pattern 구현 (DBObserver, UIObserver)
- ✅ 바 드래그 시 Duration 자동 업데이트
- ✅ `diff` 값 기반 Duration 계산
- ✅ 이벤트 기반 DB 저장 및 UI 업데이트
- ✅ UTC 날짜 처리 개선

**작동 방식**:
```
바 드래그 → update-task intercept → diff 계산 → handleTaskUpdate
→ task:updated 이벤트 발생 
→ DBObserver (DB 저장) + UIObserver (UI 업데이트)
→ Successor 재계산
```

**해결된 문제**:
1. **바 드래그 시 Duration 업데이트 안 됨**
   - 원인: `onTaskUpdate`가 드래그 이벤트를 캐치하지 못함
   - 해결: `api.intercept("update-task")` 추가

2. **Duration 계산 오류**
   - 원인: `end` 날짜만 전달되어 duration 계산 불가
   - 해결: `diff` 값을 사용한 duration 계산

3. **타임존 변환 문제**
   - 원인: `parseToUTC`가 `+00:00` 형식을 UTC로 인식하지 못함
   - 해결: UTC 형식 감지 로직 개선

4. **Duration 덮어쓰기 문제**
   - 원인: `diff`로 계산한 duration을 `end` 날짜 기반 계산이 덮어씀
   - 해결: `task.duration`이 이미 설정된 경우 재계산 건너뛰기

**테스트 결과**:
- ✅ 바 드래그로 Duration 변경 → DB 저장 확인
- ✅ 페이지 새로고침 후 Duration 유지 확인
- ✅ Event 발생 및 Observer 호출 확인
- ✅ Successor 재계산 트리거 확인

**아키텍처 개선**:
- 기존: 명령형, 산재된 업데이트 로직
- 개선: 이벤트 기반, 관심사 분리 (DB/UI/Scheduling)
- 장점: 유지보수성 향상, 실시간 협업 준비, 테스트 용이성
