# 구현 계획 (Implementation Plan)

이 문서는 기술적 의존성과 아키텍처 계층화를 기반으로 ZeroPM 기능 구현의 논리적 순서를 설명합니다.

## Goal Description
ZeroPM 기능 구현을 위한 명확한 실행 경로를 정의하여, 의존성 있는 기능(예: 일정 관리, 보고서)을 개발하기 전에 필수적인 기본 구성 요소(인증, 데이터 레이어)가 구축되도록 보장합니다.

## Phased Implementation Strategy

### Phase 1: Foundation & Data Layer (Auth + WBS + CRDT)
**Goal:** 데이터베이스와 실시간 협업 구조를 구축합니다.
*   **Dependencies:** None.
*   **Rationale:** RLS(Row Level Security)을 위한 인증이 필요합니다. WBS 데이터 구조는 Yjs(CRDT)를 사용하여 설계되어야 합니다.

1.  **Auth & Security**
    *   [x] Implement Supabase Auth (Email/Password, Social OAuth)
    *   [ ] Set up Project Member Invitation system
    *   [x] Configure RLS policies (Basic owner access)
2.  **Collaboration (Data Core)**
    *   [ ] Integrate Yjs with Supabase Realtime
    *   [x] Define Renewable Resources (Schema only)
    *   [x] Implement Infinite Hierarchy WBS (Schema & Yjs Map structure)
    *   [x] Implement Optimistic Locking (versioning)

### Phase 2: Core Logic Engine (Scheduling)
**Goal:** Gantt 차트를 렌더링하기 위한 클라이언트 측 스케줄링 엔진을 구현합니다.
*   **Dependencies:** Phase 1 (WBS Structure).
*   **Rationale:** Gantt 차트 UI는 엔진에서 ES, EF, LS, LF를 계산하여 의미 있는 데이터를 렌더링하기 위해 엔진에서 계산된 날짜를 필요로 합니다.

1.  **Scheduling Engine**
    *   [x] Implement DAG Modeling (Graphlib)
    *   [x] Implement Dependency Management (FS, SS, FF, SF, Lag/Lead)
    *   [x] Implement Cycle Detection
    *   [x] Implement Business Calendar (Workdays/Holidays)
    *   [x] Implement CPM Calculation (Forward/Backward Pass, Float, Critical Path)
    *   [x] Optimize with Web Workers

**상세 계획**: [Phase2-스케줄링엔진-구현계획.md](./Phase2-스케줄링엔진-구현계획.md)

*   **Dependencies:** Phase 1 (Data), Phase 2 (Engine Results).
*   **Rationale:** UI 컴포넌트는 Phase 1의 원시 데이터와 Phase 2의 계산된 스케줄 모두 필요합니다.

    *   [ ] Implement Optimistic UI updates
    *   [ ] Implement Presence for soft locking

*   **Dependencies:** Phase 3 (UI interactions).

## Critical Path Dependencies
- **Auth -> RLS**: Security cannot be enforced without Auth.
- **Yjs -> WBS**: The data model for tasks must be CRDT-compatible.
- **WBS -> Scheduling**: The engine needs a graph structure to calculate.
- **Scheduling -> Gantt UI**: The UI cannot render bars without start/end dates.
- **Scheduling -> Resource**: Resource usage is calculated based on time distribution.
