# Implementation Plan - Comprehensive Project Details Management

프로젝트 상세 정보 관리 기능을 **3단계(Phase)**로 나누어 구현합니다.

## 현재 상태 분석

### ✅ 이미 구현된 기능 (Schema 존재)
- **프로젝트 기본**: `id`, `title`, `owner_id`, `created_at`
- **캘린더 설정**: `calendar_settings` (JSONB)
- **작업 관리**: WBS 계층(`parent_id`), CPM 결과 필드들
- **자원 관리**: `resources`, `assignments` 테이블
- **비용 관리**: `costs` 테이블
- **기준선**: `baselines` 테이블 (확인 필요)

### ❌ 누락된 필드 (추가 필요)
**Projects 테이블:**
- `description` - 프로젝트 설명
- `start_date` - 프로젝트 시작일
- `target_end_date` - 목표 완료일
- `status` - 프로젝트 상태
- `updated_at` - 최종 수정일

export const CreateProjectSchema = z.object({
    title: z.string().min(1, '프로젝트 이름을 입력해주세요.'),
    description: z.string().optional(),
    start_date: z.string().optional(), // ISO date string
    target_end_date: z.string().optional(),
    status: z.enum(['planning', 'active', 'completed', 'on-hold', 'cancelled']).default('planning'),
});

export const UpdateProjectSchema = z.object({
    title: z.string().min(1, '프로젝트 이름을 입력해주세요.'),
    description: z.string().optional(),
    start_date: z.string().optional(),
    target_end_date: z.string().optional(),
    status: z.enum(['planning', 'active', 'completed', 'on-hold', 'cancelled']).optional(),
});

export interface Project {
    id: string;
    title: string;
    description?: string;
    start_date?: string;
    target_end_date?: string;
    status: 'planning' | 'active' | 'completed' | 'on-hold' | 'cancelled';
    owner_id: string;
    created_at: string;
    updated_at?: string;
    calendar_settings: any;
}
