# ZeroPM 화면흐름도

> 주요 화면 간 이동 경로와 인터랙션 흐름을 시각적으로 정의한다.
> 각 플로우의 상세 단계·에러 케이스는 [`03-User-Flow-시나리오.md`](../../01-유저스토리/03-User-Flow-시나리오.md) 참조.

---

## 1. 전체 화면 구조 (Information Architecture)

```mermaid
graph TD
    Landing["랜딩 페이지\n(/)"]
    Signup["회원가입\n(/auth/signup)"]
    Login["로그인\n(/auth/login)"]
    EmailConfirm["이메일 인증\n(/auth/confirm)"]
    Dashboard["대시보드\n(/dashboard)"]
    Gantt["간트 뷰\n(/projects/[id])"]
    ShareView["공유 읽기 전용\n(/share/[token])"]
    InviteAccept["초대 수락\n(/invite?token=...)"]

    Landing -->|무료로 시작| Signup
    Landing -->|로그인| Login
    Signup -->|이메일/비밀번호| EmailConfirm
    Signup -->|OAuth 완료| Dashboard
    Login -->|성공| Dashboard
    EmailConfirm -->|인증 완료 자동 로그인| Dashboard
    Dashboard -->|프로젝트 선택| Gantt
    Dashboard -->|새 프로젝트 생성| Gantt
    InviteAccept -->|수락| Gantt

    subgraph "비로그인 접근"
        ShareView
        InviteAccept
    end
```

---

## 2. 라우팅 구조

| 경로 | 화면 | 인증 필요 |
|------|------|----------|
| `/` | 랜딩 페이지 | ✗ |
| `/auth/login` | 로그인 | ✗ |
| `/auth/signup` | 회원가입 | ✗ |
| `/auth/confirm` | 이메일 인증 확인 | ✗ |
| `/invite?token=...` | 초대 수락 | △ (수락 후 로그인 유도) |
| `/dashboard` | 프로젝트 목록 | ✓ |
| `/projects/[id]` | 간트 뷰 | ✓ |
| `/share/[token]` | 공유 읽기 전용 | ✗ |

## 3. 플로우별 화면 흐름도

### 3-1. 온보딩 플로우 (Flow 1)

```mermaid
flowchart TD
    A["랜딩"] -->|무료로 시작| B["회원가입"]
    A -->|로그인| C["로그인"]

    B -->|이메일 가입| D["이메일 인증 대기"]
    B -->|OAuth| G
    D -->|이메일 링크 클릭| E["/auth/confirm"]
    E -->|자동 로그인| G

    C -->|성공| G["대시보드\n(빈 상태)"]

    G -->|새 프로젝트 만들기| H["프로젝트 생성 모달"]
    G -->|샘플로 둘러보기| I["샘플 간트 뷰\n(읽기 전용)"]
    H -->|생성 완료| J["간트 뷰\n(빈 상태)"]

    J -->|"+ 태스크 추가\n또는 빈 행 더블클릭"| K["인라인 태스크 입력"]
    K -->|"Enter 반복"| L["간트 뷰\n(태스크 2개 이상)"]
    L -->|의존성 연결 안내 툴팁 표시| L
```

### 3-2. 프리랜서 PM 일간 작업 플로우 (Flow 2)

```mermaid
flowchart LR
    A["대시보드"] -->|프로젝트 카드 클릭| B["간트 뷰"]

    B -->|태스크 바 클릭| C["태스크 상세 패널"]
    B -->|태스크 바 드래그| D["드래그 중\n날짜 툴팁 표시"]

    C -->|시작일 수정| E["자동 전파\n후행 태스크 재계산"]
    D -->|드롭 확정| E

    E -->|전파 완료\n배너 3초 표시| B

    B -->|공유 버튼| F["공유 설정 패널"]
    F -->|링크 복사| G["클립보드 저장\n토스트 표시"]
```

### 3-3. 팀 PM 협업 플로우 (Flow 3)

```mermaid
flowchart TD
    A["간트 뷰\n(실시간 접속 중)"]

    A -->|팀원 태스크 클릭| B["태스크 상세 패널"]
    B -->|진행률 100% 설정| C["완료 처리"]
    C -->|Supabase Realtime| D["팀원 화면 실시간 반영"]
    C -->|CP 재계산| E["상단 요약 갱신\n(Float, 크리티컬 수)"]

    A -->|멤버 관리 아이콘| F["멤버 관리 패널"]
    F -->|이메일 + 역할 입력| G["초대 발송\n(이메일 전송 + pending 표시)"]
    G --> A
```

### 3-4. 충돌 해결 플로우 (Flow 4)

```mermaid
flowchart TD
    A["간트 뷰"] -->|태스크 수정 저장 시도| B{409\nConflict?}

    B -->|아니오| C["저장 완료\n'저장됨' 상태 칩"]
    B -->|예| D["충돌 감지 모달\n(변경 내용 비교표)"]

    D -->|상대방 변경 유지| E["최신 값 적용"]
    D -->|내 변경 적용| F["강제 저장\n상대방 화면에 알림"]
    D -->|항목별 선택| G["병합 모달\n라디오 버튼 선택"]
    D -->|나중에 결정| H["충돌 미해결\n태스크 잠금\n툴바 배지 표시"]

    E --> A
    F --> A
    G -->|병합 저장| A
    H -->|툴바 배지 클릭| D
```

### 3-5. Undo/Redo 플로우 (Flow 5)

```mermaid
flowchart LR
    A["간트 뷰"] -->|편집 액션\n드래그·생성·삭제·링크| B["Undo 스택 누적\n(그룹 단위, 최대 50)"]
    B --> A

    A -->|"Ctrl+Z / Undo 버튼"| C["이전 상태 복원\n(그룹 전체 되돌림)"]
    C -->|복원 애니메이션 0.3초| A
    C -->|"토스트: 'N개 변경 취소됨 [다시 실행]'"| A

    A -->|"Ctrl+Y / Redo 버튼"| D["재실행"]
    D --> A
```

### 3-6. 오프라인 → 온라인 복귀 플로우 (Flow 6)

```mermaid
flowchart TD
    A["간트 뷰\n(온라인)"] -->|네트워크 끊김| B["오프라인 배너 표시\n상단 전체 너비"]
    B -->|계속 편집| C["로컬 변경 대기열 누적"]
    C -->|네트워크 복귀 감지| D{충돌\n있음?}

    D -->|없음| E["자동 동기화 완료\n'N개 변경 저장됨' 배너 3초"]
    D -->|있음| F["오프라인 충돌 모달\n(Flow 4와 동일 UX)"]

    E --> A
    F -->|충돌 해결| E
```

### 3-7. 팀원 초대 플로우 (Flow 7)

```mermaid
flowchart TD
    A["간트 뷰\n(Owner 권한)"] -->|멤버 관리 패널 열기| B["이메일 입력 + 역할 선택"]
    B -->|초대 보내기| C["초대 이메일 발송\npending 배지 표시"]

    C -->|수신자: 이메일 링크 클릭| D["/invite?token=..."]

    D -->|기존 가입자 + 로그인 상태| E["참여 확인 화면\n프로젝트 열기 버튼"]
    D -->|비로그인 상태| F["로그인 페이지\n(redirect 파라미터 유지)"]
    D -->|미가입 신규 사용자| G["회원가입\n(이메일 자동 입력·잠금)"]
    D -->|만료된 토큰 (48시간 초과)| H["'링크가 만료됐어요'\n(재초대 요청 안내)"]
    D -->|다른 계정 로그인 상태| I["'초대받은 이메일로 로그인하세요'\n(로그아웃 후 재로그인 버튼)"]

    F -->|로그인 완료| E
    G -->|가입 완료| E
    E -->|프로젝트 열기| J["간트 뷰\n(역할 권한 적용됨)"]
```

### 3-8. 기준선 저장·비교 플로우 (Flow 8)

```mermaid
flowchart TD
    A["간트 뷰"] -->|기준선 드롭다운| B["기준선 저장 다이얼로그\n(이름 입력, 최대 11개)"]
    B -->|저장| C["DB 스냅샷 저장\n토스트 표시"]
    C -->|저장 즉시 활성화| D["기준선 비교 뷰\n(현재 바 + 기준선 바 병렬 표시)"]

    A -->|드롭다운에서 기준선 선택| D

    D -->|지연 태스크 클릭| E["태스크 상세 패널\n(슬리피지 +N일 표시)"]
    D -->|다른 기준선 선택| D
    D -->|"[✕] 비교 종료"| A

    D -->|기준선 삭제 클릭| F{현재\n비교 중?}
    F -->|예| G["확인 다이얼로그"] -->|삭제 확인| A
    F -->|아니오| H["즉시 삭제\n'실행취소' 토스트 5초"] --> D
```

### 3-9. 공유 링크 플로우 (Flow 9)

```mermaid
flowchart LR
    A["간트 뷰"] -->|공유 버튼| B["공유 설정 패널"]
    B -->|토글 ON| C["링크 생성됨\n(고유 토큰 포함 URL)"]
    C -->|복사 버튼| D["클립보드 복사\n'복사됐습니다' 토스트"]

    C -->|수신자 URL 접속| E["공유 읽기 전용\n(/share/[token])"]
    E -->|편집 시도| F["인터랙션 차단\n(커서 금지, 버튼 disabled)"]

    B -->|토글 OFF| G["링크 비활성화\n기존 토큰 즉시 무효화"]
    G -->|수신자 기존 링크 접속| H["'이 링크는 만료됐습니다'"]

    B -->|새 링크 생성| I["기존 토큰 무효화\n신규 토큰 발급"]
    I --> C
```

## 4. 간트 뷰 내부 화면 구조

간트 뷰(`/projects/[id]`)는 ZeroPM의 주 작업 화면으로, 레이아웃과 오버레이로 구성된다.

```mermaid
graph TD
    subgraph GanttView["간트 뷰 (/projects/[id])"]
        Toolbar["상단 툴바\n[+태스크] [Undo] [Redo] [기준선▼] [공유] [멤버]"]
        WBS["좌측 WBS 그리드\n태스크 목록 + 인라인 편집"]
        Timeline["우측 타임라인\n간트 바 + 링크 + 기준선 바"]
        StatusBar["상단 상태 칩\n저장 중... / 저장됨 / 오프라인 배너"]
    end

    subgraph Overlays["패널 & 오버레이"]
        TaskPanel["태스크 상세 패널\n(우측 슬라이드)"]
        MemberPanel["멤버 관리 패널\n(슬라이드 오버)"]
        SharePanel["공유 설정 패널"]
        BaselineDD["기준선 드롭다운"]
        ConflictModal["충돌 해결 모달"]
        UndoTooltip["Undo 호버 툴팁\n(되돌릴 작업 설명)"]
    end

    WBS -->|행 클릭| TaskPanel
    Timeline -->|바 클릭| TaskPanel
    Toolbar -->|멤버 아이콘| MemberPanel
    Toolbar -->|공유 버튼| SharePanel
    Toolbar -->|기준선 버튼| BaselineDD
    Toolbar -->|Undo 버튼 호버| UndoTooltip
    Timeline -->|충돌 발생| ConflictModal
```

## 5. 권한별 화면 차이

| UI 요소 | Viewer | Editor | Owner |
|---------|--------|--------|-------|
| 간트 조회 | ✅ | ✅ | ✅ |
| 태스크 드래그·리사이즈 | 커서 금지 | ✅ | ✅ |
| 태스크 추가·삭제 | 버튼 비활성 | ✅ | ✅ |
| 링크(의존성) 생성·삭제 | 불가 | ✅ | ✅ |
| 기준선 저장 | 불가 | ✅ | ✅ |
| 멤버 관리 패널 | 미노출 | 목록 조회만 | 초대·제거 가능 |
| 프로젝트 설정·삭제 | 불가 | 불가 | ✅ |

> 태스크 상세 패널은 Viewer도 열 수 있으나 모든 입력 필드가 disabled 상태로 표시되며, 패널 하단에 "읽기 전용 — Viewer 권한" 레이블이 표시된다.
