## ZeroPM 최종 통합 상세 기능 목록 (Final Comprehensive Feature List)

### I. 인증, 사용자 관리 및 접근 제어 (Authentication, User Management & Security)

ZeroPM은 **Supabase Auth**를 통해 인증 기능을, **Supabase RLS**를 통해 데이터 접근 제어를 구현합니다.

| 기능 분류 | 상세 기능 | 설명 및 기술적 근거 |
| :--- | :--- | :--- |
| **인증 (Authentication)** | **이메일/비밀번호 로그인 및 회원가입** | 기본적인 사용자 계정 생성 및 로그인 기능을 제공합니다. Supabase Auth의 내장 기능을 활용합니다. |
| | **소셜 OAuth 로그인 지원** | Google, GitHub 등 소셜 로그인 공급자(Social OAuth providers)를 통한 인증을 지원합니다. |
| **권한 및 접근 제어** | **행 레벨 보안 (RLS) 적용** | **PostgreSQL Row Level Security (RLS)**를 활성화하여, 인증된 사용자(authenticated)가 자신이 속한 프로젝트의 데이터에만 접근하고 수정할 수 있도록 데이터베이스 레벨에서 접근을 통제합니다. |
| | **역할 기반 접근 제어 (RBAC) 기반** | 소규모 팀 운영을 위해 사용자 프로필에 `admin`, `member` 등의 역할을 정의하고, RLS 정책에서 이를 활용하여 권한을 세분화합니다. |
| **사용자 관리** | **MAU 기반 무료 사용** | Supabase 무료 티어는 월 **50,000 MAUs (Monthly Active Users)**를 포함하므로, 소규모 팀 및 개인 사용에는 비용 걱정 없이 사용자 관리가 가능합니다. |
| | **프로젝트 멤버 초대 및 관리** | 프로젝트 소유자(owner\_id)가 팀 멤버를 초대하고 역할을 할당하는 기능을 지원해야 합니다. |

### II. 실시간 협업 및 데이터 무결성 (Collaboration & Integrity)

| 기능 분류 | 상세 기능 | 설명 및 기술적 근거 |
| :--- | :--- | :--- |
| **실시간 동시 편집** | **CRDT 기반 동기화 (Yjs + Realtime)** | **Yjs (CRDT)**와 **Supabase Realtime**을 통합하여 여러 사용자의 동시 편집 시 데이터 충돌 없이 자동으로 병합되도록 구현합니다. Supabase 무료 티어는 월 **200만 건의 메시지**와 **200개의 동시 연결**을 지원합니다. |
| **현재 사용자 인식** | **프레즌스 (Presence) 기반 소프트 락** | Supabase **Presence** 기능을 사용하여 다른 사용자가 현재 **어떤 Task를 편집 중인지** 실시간으로 시각화하여 논리적 충돌을 방지합니다. |
| **클라이언트 우선 UX** | **Optimistic UI 패턴 및 오프라인 지원** | 사용자 액션(예: Task 드래그)에 대해 즉시 UI를 업데이트하고, **IndexedDB** 또는 **RxDB** 등을 활용한 로컬 데이터베이스에 CRDT 상태를 영구 저장하여 오프라인 작업 후 자동 동기화를 지원합니다. |
| **동시 쓰기 처리** | **낙관적 잠금 (Optimistic Locking)** | Task 데이터에 `version` 필드를 사용하여, CRDT가 담당하지 않는 구조적 변경(삭제, 구조 이동) 시 **낙관적 잠금** 패턴을 적용하여 데이터 무결성을 보장합니다. |
| **트랜잭션 관리** | **PostgreSQL 트랜잭션 활용** | Task, Dependency, Resource 할당 등 논리적으로 연결된 여러 데이터 변경을 **원자적(Atomic) 단위**로 처리하기 위해 PostgreSQL의 **트랜잭션** (`BEGIN`, `COMMIT`, `ROLLBACK`) 기능을 활용합니다. 이는 Supabase의 RPC 호출을 통해 구현할 수 있습니다. |

### III. 핵심 프로젝트 관리 및 WBS (PM Core & WBS)

| 기능 분류 | 상세 기능 | 설명 및 기술적 근거 |
| :--- | :--- | :--- |
| **WBS 구조 관리** | **무한 계층형 WBS 지원** | Task의 계층 구조를 PostgreSQL의 **인접 리스트(Adjacency List)** 모델 및 **재귀 CTE**를 통해 지원하여, 깊은 WBS를 관리합니다. |
| **요약 작업 자동화** | **Roll-up 기능** | 하위 Task의 기간, 완료율, 비용 등이 상위 요약 작업(Summary Task)으로 자동 집계됩니다. |
| **태스크 유형 확장** | **Custom Task Type 지원** | 기본 Task, Milestone, Summary 외에 `urgent`, `narrow`, `progress` 등 사용자 정의 Task Type을 추가하고 CSS로 스타일을 적용할 수 있습니다. 이는 **SVAR React Gantt**에서 지원하는 기능입니다. |
| **기준선 관리** | **다중 기준선 (Baseline) 설정** | 프로젝트의 원래 계획(범위, 일정, 비용)을 고정된 참조점(Fixed reference point)으로 저장합니다. 최대 **11개**의 기준선 데이터셋을 저장하고 관리할 수 있습니다. |
| **진행 상황 스냅샷** | **중간 계획 (Interim Plan) 설정** | 프로젝트 진행 중 현재의 시작일 및 종료일 스냅샷을 최대 **10개**까지 저장할 수 있습니다. |

### IV. 고급 스케줄링 및 엔진 (Advanced Scheduling Engine)

클라이언트 측 JavaScript/TypeScript 환경에서 자체 개발된 엔진이 고성능으로 동작해야 합니다.

| 기능 분류 | 상세 기능 | 설명 및 기술적 근거 |
| :--- | :--- | :--- |
| **그래프 모델링** | **Graphlib 기반 DAG 구성** | **@dagrejs/graphlib**와 같은 라이브러리를 활용하여 Task와 Dependency를 노드와 엣지로 모델링합니다. |
| **유효성 검증** | **순환 참조(Cycle) 감지** | 의존성 그래프에 순환(Cycle)이 발생할 경우, **위상 정렬(Topological Sort, Kahn's algorithm 또는 DFS)** 알고리즘을 사용하여 즉시 감지하고 사용자에게 경고해야 합니다. |
| **의존성 및 제약** | **4가지 의존성 유형** | **FS, SS, FF, SF** 모두 지원하며, **Lag** 및 **Lead Time**을 적용합니다. |
| **CPM 계산** | **전진/후진 계산** | **CPM**을 통해 조기 시작일(ES), 조기 종료일(EF), 늦은 시작일(LS), 늦은 완료일(LF)을 계산하고, **Total Float**을 도출합니다. |
| **임계 경로 시각화** | **Critical Path 하이라이팅** | Total Float이 0인 작업을 식별하여 Gantt 차트에서 시각적으로 강조합니다. |
| **업무일 기반 계산** | **비즈니스 캘린더 반영** | `dayjs-business-days`와 같은 라이브러리를 채택하여 주말 및 사용자 정의 **공휴일(Holidays)**을 제외한 유효 작업 시간만을 계산합니다. |
| **성능 최적화** | **Web Worker를 활용한 비동기 계산** | CPM 및 RCPSP 계산은 **Web Worker**를 통해 백그라운드에서 비동기적으로 실행되어 UI 메인 스레드의 블로킹을 방지합니다. |

### V. 자원 및 비용 관리 (Resource & Cost Management)

| 기능 분류 | 상세 기능 | 설명 및 기술적 근거 |
| :--- | :--- | :--- |
| **자원 모델링** | **갱신 가능한 자원 (Renewable Resources) 정의** | 인력(People), 장비(Equipment) 등 시간 단위마다 가용량이 제한되는 자원($R_k$)을 정의하고, 각 작업($j$)의 자원 요구량($r_{jk}$)을 설정합니다. |
| **자원 할당** | **할당 단위 설정** | Task에 자원을 50%, 100% 등의 할당 단위로 설정할 수 있어야 합니다. |
| **RCPSP 충돌 감지** | **실시간 과부하 감지 및 시각화** | 특정 시점의 자원 요구량 합계($\sum r_{jk}$)가 가용량($R_k$)을 초과하는지 **실시간으로 감지**하고 UI에 시각적으로 표시합니다. |
| **휴리스틱 지원** | **조정 우선순위 추천** | 자원 충돌 시 수동 조정을 돕기 위해 **LFT(Latest Finish Time)** 또는 **GRD(Greatest Resource Demand)**와 같은 휴리스틱 기반의 우선순위 정보를 제공합니다. |
| **비용 데이터 입력** | **시간당 요율 및 고정 비용** | 작업에 투입되는 **시간당 비용(Rate)** 및 **고정 비용(Fixed Cost)**을 입력할 수 있어야 합니다. |
| **비용 성과 분석** | **기준선 비교 및 편차 분석** | 설정된 **비용 기준선(Cost Baseline)** 대비 현재 비용 편차(CV) 및 **비용 성과 지표(CPI)**를 계산하여 프로젝트의 재정 상태를 진단합니다. |

### VI. 사용자 인터페이스 및 아키텍처 (UI/Architecture)

| 기능 분류 | 상세 기능 | 설명 및 기술적 근거 |
| :--- | :--- | :--- |
| **고성능 Gantt** | **가상화 (Virtualization) 렌더링** | **SVAR React Gantt**를 사용하여 행 및 타임라인 가상화 기술을 적용, 수천 개 Task를 끊김 없이 렌더링합니다. |
| **보조 시각화** | **Kanban 및 Calendar View** | Task의 상태 또는 날짜를 중심으로 시각화하는 Kanban Board 및 Calendar View (FullCalendar 코어 활용 권장)를 제공합니다. |
| **데이터 그리드** | **가상화된 Task 데이터 그리드** | Gantt 차트의 좌측 Task 목록 영역(데이터 그리드)에도 **가상화**를 적용하여 대용량 데이터의 편집 및 스크롤 성능을 확보해야 합니다. |
| **기술 스택** | **Next.js / Supabase / Vercel** | 프론트엔드는 Next.js + Vercel Hobby Plan (Zero-Cost 호스팅)으로, 백엔드는 Supabase Free Tier (PostgreSQL + Auth)로 구축합니다. |
