## MoSCoW 기능 우선순위

> **스토리 맵 참조 표기법**: `A#/E#-#` = Activity 번호 / Epic 번호. 예: `A2/E2-2` = Activity 2 (일정 계획) > Epic 2-2 (의존성 및 자동 스케줄링). 전체 Activity/Epic 목록은 [01-페르소나-사용자스토리맵.md](01-페르소나-사용자스토리맵.md) 참고.

## Phase별 재정의 로드맵

---

### Phase 1 — "혼자 써도 가치 있는 도구" (현재 → Closed Beta)

**목표**: 프리랜서 PM 1인이 실제 클라이언트 프로젝트를 ZeroPM으로 관리하고 "MS Project 없이도 되겠다"고 느끼는 첫 경험을 제공한다.

**Entry Criteria**
- 개발 환경(Next.js + Supabase)이 구성되어 있고 CI/CD 파이프라인이 동작
- SVAR 간트 기반 렌더링이 로컬에서 안정적으로 작동
- 핵심 개발자가 풀타임으로 투입 가능한 상태

**Exit Criteria**
- 신규 사용자가 가입 후 5분 내에 간트 차트가 포함된 프로젝트를 생성할 수 있음 (M-09 + S-08)
- FS 의존성 링크 생성 시 후행 태스크 날짜가 즉시 자동 재계산됨 (수동 조작 없음)
- 크리티컬 패스가 시각적으로 구분되어 표시됨
- 기준선 1개 이상 저장 및 조회 가능
- 5개 이상의 외부 베타 사용자가 실제 프로젝트 데이터를 입력하고 이탈 없이 2주 이상 사용
- 치명적(P0) 버그 0건, 주요(P1) 버그 3건 이하

**핵심 기능 목록**
- M-01 프로젝트/WBS CRUD
- M-02 간트 차트 (드래그/리사이즈/링크)
- M-03 의존성 자동 스케줄링 (4종 링크 + Lag)
- M-04 제약 조건 6종
- M-05 크리티컬 패스 시각화
- M-06 Auth + RLS
- M-08 기준선 저장 (저장만; 비교 뷰는 Phase 2에서 UI 완성)
- M-09 온보딩 UX (회원가입·이메일 인증 플로우·빈 대시보드 진입)
- S-07 줌 레벨 (일/주/월)
- S-08 온보딩 가이드 (빈 상태 CTA·간트 툴팁·샘플 프로젝트)

**예상 소요**: M (현재 기능 기반, 안정화 + UX 폴리싱 중심)

### Phase 2 — "팀이 함께 쓸 수 있는 도구" (Closed Beta → Open Beta)

**목표**: IT 팀 PM이 5~15명 팀원과 함께 스프린트/릴리즈 일정을 ZeroPM에서 관리하며 "Jira 타임라인보다 낫다"는 경험을 제공한다.

**Entry Criteria**
- Phase 1 Exit Criteria를 모두 충족
- 최소 10개 이상의 실제 프로젝트 데이터(태스크 30개 이상)가 프로덕션에 존재
- P0/P1 버그가 모두 해소된 상태

**Exit Criteria**
- 3인 이상이 동시에 같은 프로젝트를 편집해도 데이터 충돌 없이 동기화됨
- 멤버 초대 후 역할(Viewer/Editor)에 따라 편집 권한이 정확히 제한됨
- 공유 링크(섹션 단위 포함)로 외부 파트너가 지정된 범위만 열람 가능
- Undo/Redo가 최소 20단계 작동하고 간트 상태가 올바르게 복원됨
- 기준선 비교 뷰에서 계획 대비 실적 슬리피지가 시각적으로 표시됨
- 태스크 변경 시 영향받는 후행 태스크 수가 확인 메시지로 표시됨
- 태스크 완료율(0~100%)을 입력하면 간트 바 내부에 즉시 진행 표시가 나타남
- 태스크에 코멘트를 입력하면 같은 프로젝트 팀원에게 실시간으로 반영됨
- 5개 이상 팀(팀당 3인 이상)이 2주 이상 지속 사용
- WAU 증가율이 2주 연속 10% 이상

**핵심 기능 목록**
- M-07 실시간 멀티유저 동기화 (안정화)
- M-08 기준선 비교 뷰 (UI 완성; 저장은 Phase 1에서 완료)
- S-01 Undo/Redo
- S-02 태스크 완료율(%) 입력 및 표시
- S-05 태스크 코멘트/메모
- S-06 멤버 초대 및 역할 관리
- S-10 스프레드시트식 붙여넣기 일괄 태스크 입력
- S-11 일정 변경 영향 범위 확인 메시지
- S-12 섹션 단위 공유 링크

**예상 소요**: L (Undo/Redo가 간트 인터셉트 레이어와 통합되어야 하므로 가장 난이도 높음)

### Phase 3 — "보고하고 공유할 수 있는 도구" (Open Beta → v1.0 정식 출시)

**목표**: 프리랜서 PM이 클라이언트에게 대시보드 현황 또는 PDF를 전달하고, IT 팀 PM이 경영진에게 프로젝트 현황을 대시보드로 보여줄 수 있다. (공유 링크 자체는 Phase 2에서 완성)

**Entry Criteria**
- Phase 2 Exit Criteria를 모두 충족
- MAU 100명 이상 달성
- 사용자 인터뷰에서 "보고 기능 필요"가 Top 3 피드백으로 확인됨

**Exit Criteria**
- 프로젝트 대시보드에서 전체 태스크 수, 완료율, 크리티컬 패스 길이, 기준선 편차가 한 화면에 표시됨
- 간트 차트를 PDF로 내보내기 했을 때 레이아웃이 깨지지 않음
- CSV로 내보낸 파일을 재가져오기 했을 때 태스크/링크 데이터가 정확히 복원됨
- 담당자가 배정된 태스크가 간트에 시각적으로 표시됨
- 프로젝트 템플릿(소프트웨어 출시 등)으로 신규 프로젝트를 5분 내에 생성 가능
- 사용자 만족도(CSAT) 4.0/5.0 이상
- p95 응답 시간이 NFR 기준 이하 유지

**핵심 기능 목록**
- S-03 프로젝트 대시보드 (요약 뷰)
- S-04 PDF 내보내기
- S-09 CSV 내보내기/가져오기
- C-01 담당자 배정 (태스크별 담당자 표시; 현황 뷰는 Phase 4)
- C-04 프로젝트 템플릿 (소프트웨어 출시 2종, 이벤트 기획 1종)
- C-05 마감 임박 알림 (이메일)

**예상 소요**: L

### Phase 4 — "성장하는 팀을 위한 도구" (v1.0 이후)

**목표**: 팀 규모가 15~30명으로 늘어도 ZeroPM이 버티며, 커스텀 필드·활동 로그·시나리오 시뮬레이션으로 고급 프로젝트 관리를 지원한다.

**Entry Criteria**
- Phase 3 Exit Criteria를 모두 충족
- MAU 500명 이상, 유료 전환 실험을 위한 결제 인프라 준비 완료

**Exit Criteria**
- 태스크 500개, 동시 접속 20명 시나리오에서 NFR 성능 기준 충족
- 자원별 배정 현황 뷰(히트맵 또는 바 차트)가 작동
- 변경 이력 로그에서 최근 100건 조회가 2초 이내
- 커스텀 필드 최소 3종(텍스트/숫자/날짜) 생성 및 간트 열 표시 가능
- What-if 시뮬레이션에서 현재 일정을 보존한 채 가상 시나리오를 독립적으로 비교 가능

**핵심 기능 목록**
- C-01 자원 배정 현황 뷰 (히트맵·바 차트; 담당자 표시는 Phase 3에서 완료)
- C-02 태그/커스텀 필드
- C-03 활동 로그 (변경 이력)
- C-06 다크 모드
- C-07 What-if 시나리오 시뮬레이션

**예상 소요**: XL
