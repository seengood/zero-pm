# Supabase 데이터베이스 마이그레이션 가이드

## 방법 1: Supabase Dashboard 사용 (권장)

### 단계 1: Supabase Dashboard 접속

1. https://supabase.com/dashboard 접속
2. 프로젝트 선택: `rszftuyndidleofydsbb`
3. 왼쪽 메뉴에서 **SQL Editor** 클릭

### 단계 2: 스키마 실행

**⚠️ 중요: 기존 테이블이 있는 경우**

기존에 `projects`, `tasks`, `links` 등의 테이블이 이미 있다면:
- ❌ **전체 `schema.sql`을 실행하면 안 됩니다** (오류 발생)
- ✅ **`migration-incremental.sql`을 사용하세요** (안전한 증분 마이그레이션)

#### 방법 A: 기존 테이블이 있는 경우 (권장)

1. **New Query** 버튼 클릭
2. `supabase/migration-incremental.sql` 파일의 내용을 복사하여 붙여넣기
3. **Run** 버튼 클릭하여 실행

이 스크립트는:
- 새로운 `user_profiles` 테이블만 생성
- 기존 RLS 정책을 안전하게 업데이트 (DROP IF EXISTS 사용)
- observer 역할 지원 추가
- 낙관적 잠금 및 트랜잭션 함수 추가

#### 방법 B: 새로운 프로젝트인 경우

1. **New Query** 버튼 클릭
2. `supabase/schema.sql` 파일의 내용을 복사하여 붙여넣기
3. **Run** 버튼 클릭하여 실행

### 단계 3: 실행 확인

1. 왼쪽 메뉴에서 **Table Editor** 클릭
2. 다음 테이블들이 생성되었는지 확인:
   - `projects`
   - `user_profiles` ⭐ (새로 추가됨)
   - `tasks`
   - `links`
   - `project_members`
   - `resources`
   - `assignments`
   - `costs`
   - `baselines`
   - `baseline_tasks`
   - `interim_plans`

### 단계 4: RLS 정책 확인

1. 각 테이블을 클릭
2. **Policies** 탭 확인
3. RLS가 활성화되어 있고 정책들이 생성되었는지 확인

## 검증 방법

### SQL Editor에서 확인

다음 쿼리를 실행하여 테이블 목록 확인:

```sql
SELECT table_name 
FROM information_schema.tables 
WHERE table_schema = 'public' 
ORDER BY table_name;
```

### RLS 정책 확인

```sql
SELECT schemaname, tablename, policyname, permissive, roles, cmd, qual
FROM pg_policies
WHERE schemaname = 'public'
ORDER BY tablename, policyname;
```

### 함수 확인

```sql
SELECT routine_name, routine_type
FROM information_schema.routines
WHERE routine_schema = 'public'
AND routine_name IN ('update_task_with_version', 'save_cpm_results', 'create_baseline', 'is_project_owner')
ORDER BY routine_name;
```

## 주의사항

⚠️ **기존 데이터가 있는 경우**:
- 스키마 실행 전에 백업을 권장합니다
- `user_profiles` 테이블이 새로 추가되므로 기존 사용자가 있다면 수동으로 프로필을 생성해야 합니다

⚠️ **Observer 역할**:
- 기존 `project_members`에 observer 역할이 추가되었습니다
- 기존 멤버들의 역할은 자동으로 'member'로 유지됩니다

## 문제 해결

### "relation already exists" 오류

기존 테이블이 있는 경우 발생합니다. 다음 중 하나를 선택:

1. **기존 테이블 유지**: 새로 추가된 부분만 실행
2. **초기화 후 재생성**: `supabase/reset.sql` 실행 후 `schema.sql` 실행

### RLS 정책 충돌

기존 정책이 있는 경우:

```sql
-- 기존 정책 삭제 (필요한 경우)
DROP POLICY IF EXISTS "policy_name" ON table_name;
```
